技術的に正しいバーコードを生成することは、仕事の半分に過ぎません。間違ったファイル形式でダウンロードしたり、解像度が不十分であったりすると、印刷結果がぼやけたり、ピクセル化されたり、圧縮されたりして、バーコードのスキャンに失敗します。このガイドでは、SVG、PNG、PDF バーコード形式の違いについて説明し、状況に応じてどれを使用するかを正確に示します。

3 つのバーコード ファイル形式: SVG、PNG、PDF

SVG

スケーラブルなベクター グラフィックス

形状の数学的記述。無限に拡張可能 — 1cm のラベルから看板まで、あらゆるサイズに最適です。

✓ 印刷に最適 ✓ どのようなサイズでも品質の低下はありません ✓ 最小のファイルサイズ ✗ すべてのアプリでサポートされているわけではありません
PNG

ポータブルネットワークグラフィックス

可逆圧縮によるピクセルベースの画像。品質は生成された解像度によって異なります。

✓ ユニバーサルな互換性 ✓ ロスレス — 圧縮アーティファクトなし ✓ 透明性をサポート ✗ 拡大するとピクセル化する
PDF

ポータブルドキュメントフォーマット

印刷可能なドキュメント形式。ジェネレーターによっては、ベクターまたはラスター バーコードが含まれる場合があります。

✓ 直接印刷する準備ができています ✓ 正確な寸法を維持 ✗ 他のデザインに埋め込むのが難しい ✗ 品質はソースに依存します

SVG バーコード — プロフェッショナルな印刷に最適な選択肢

SVG (Scalable Vector Graphics) はベクトル形式であり、バーコードがピクセルではなく数学的形状として記述されることを意味します。ベクター バーコードは、ラベルに 2 cm で印刷する場合でも、バナーに 2 メートルで印刷する場合でも、どのサイズでも完全にシャープでクリーンなエッジを備えています。ピクセル化、ぼやけ、アーティファクトはありません。

SVG は、プリンターや包装メーカーに提供される製品パッケージのアートワーク、Adobe Illustrator、Affinity Designer、または InDesign で使用されるラベル デザイン ファイル、バーコードをさまざまなサイズに拡大縮小できる状況、および GS1 印刷品質基準を満たす必要がある小売パッケージには正しい選択です。

業界の推奨事項: 常に SVG バーコードをパッケージ デザイナー、ラベル メーカー、印刷会社に提供してください。ベクター ファイルでは、どのように再現されたとしてもバーコードが鮮明なままであることが保証されます。

デザイン ソフトウェアで SVG バーコードを使用する方法

すべてのプロフェッショナルなデザイン ツールは SVG をサポートしています。 Adobe Illustrator で、「ファイル」→「配置」を使用して SVG ファイルを選択します。 Affinity Designer で、SVG をキャンバスに直接ドラッグ アンド ドロップします。 Canva で、SVG をカスタム要素としてアップロードします。バーコードは完全にスケーラブルで、ベクトル オブジェクトとして編集可能です。

Web 上の SVG バーコード

SVG バーコードは、img タグまたはインライン SVG コードを使用して HTML ページに直接埋め込むこともできます。 Retina ディスプレイや高 DPI ディスプレイを含むすべての画面密度で鮮明にレンダリングされ、ファイル サイズは通常、同等の PNG ファイルよりも小さくなります。

PNG バーコード — デジタル使用およびドキュメントに最適

PNG (Portable Network Graphics) はラスター形式で、画像はグリッド状に配置されたピクセルで構成されます。 PNG は可逆圧縮を使用します。つまり、JPEG とは異なり、ファイルの保存時に画像データが破棄されません。これにより、画像が十分な解像度で生成されている限り、PNG はバーコードに対して安全になります。

Word 文書、Excel スプレッドシート、Google ドキュメントへのバーコードの埋め込み、電子メールやプレゼンテーションへのバーコードの追加、Web サイトやアプリでのバーコードの表示、固定の既知のサイズで使用されるバーコード イメージには、PNG が最適です。

PNG 解像度の要件

PNG バーコードの品質は、生成された解像度に完全に依存します。印刷に安全な最小解像度は次のとおりです。 300 DPI。この解像度では、100% GS1 公称サイズ (37.29mm x 25.91mm) の UPC-A バーコードは少なくとも 440 x 306 ピクセルでなければなりません。 Code 128 または細いバーを含むその他の高密度バーコードの場合は、バーのエッジが鮮明になるように 600 DPI で生成します。

PNG バーコードを拡大しないでください。 150 DPI でバーコードを生成し、それを Word または Photoshop で拡大すると、画像がピクセル化されます。バーが柔らかくなり、鋭いエッジが失われ、バーコードのスキャンが失敗します。常に最初から必要な解像度で PNG を生成するか、代わりに SVG を使用してください。

PDF バーコード — 直接印刷に最適

PDF バーコード ファイルは、定義された物理サイズでバーコードを含む印刷可能なドキュメントです。 PDF は、バーコードをデザイン アプリケーションにインポートすることなく、コンピューターから直接印刷する場合に最も便利な形式です。 PDF を開いて 100% の倍率で印刷するだけです。サイズ変更や拡大縮小は不要で、正確な寸法が保証されます。

PDF バーコードは、デザイン ツールを使用せずに簡単に使い捨てで印刷したり、すぐに印刷する必要があるデザイナー以外の人にバーコードを送信したり、固定サイズの印刷可能なファイルを必要とするワークフローに役立ちます。ベクター PDF (バーコードがピクセルではなくパスとして記述される) は、品質において SVG と同等です。当社のバーコード ジェネレーターは、すべての 1D バーコード タイプのベクター PDF を出力します。

バーコードに JPEG を決して使用してはいけない理由

JPEG は非可逆圧縮形式であり、ファイル サイズを削減するために画像データを意図的に破棄します。この圧縮プロセスにより、高コントラストのエッジに沿ってアーティファクトが生じますが、これがまさにバーコード バーです。 JPEG 圧縮により、黒いバーと白いスペースの間の明確な境界がぼやけて不鮮明になり、バーの幅が不正確になります。バーコード スキャナはバーの幅を読み取ってデータをデコードするため、エッジがぼやけてしまうと、デコードが不正確になったり、スキャンが完全に失敗したりすることになります。

いかなる状況でもバーコードに JPEG を使用しないでください。常に SVG、PNG、または PDF を使用してください。誰かが JPEG バーコードを送ってきた場合は、それを拒否し、PNG または SVG の置き換えをリクエストしてください。

バーコード印刷の DPI 要件

DPI (ドット/インチ) は、プリンターが紙 1 インチあたりに配置するインク ドットの数を決定します。 1 インチあたりのドット数が多いほど、プリンターで再現できる細部がより細かくなります。

DPI適合性注意事項
72～96 DPI画面表示のみ標準的な画面解像度。印刷には絶対に使用しないでください。
150 DPIマージナル — バーコードの場合は避ける理想的な条件でスキャンできる可能性がありますが、信頼性は高くありません。 GS1に準拠していません。
200 DPI一部の 1D バーコードで許容される最小値十分なサイズで印刷される大きなバーコードにのみ使用してください。危険です。
300 DPIすべてのバーコードに推奨される最小値ラベル印刷とパッケージングの標準。 UPC/EAN に対して信頼性があります。
600 DPI高密度バーコードに推奨コンパクトなコード 128、データ マトリックス、および小さな QR コードに必要です。
1200以上のDPIプロフェッショナルなパッケージングと感熱印刷産業用ラベルプリンターで使用されます。最高の品質。

感熱ラベルプリンターとDPI

感熱ラベル プリンタ (倉庫、小売在庫室、出荷部門で使用されるタイプ) は、プリント ヘッドの解像度によって評価されます。一般的な評価は 203 DPI、300 DPI、および 600 DPI です。 203 DPI のサーマル プリンタは、標準ラベル ストック上の大きな Code 128 出荷用バーコードに適しています。 GS1 検証グレードを満たす必要がある小さなバーコード、データ マトリックス コード、またはラベルの場合は、300 DPI または 600 DPI のサーマル プリンタをお勧めします。

よくある質問

バーコードに最適なファイル形式は何ですか?
SVG は、ピクセル化せずに任意のサイズに拡大縮小できるベクター形式であるため、プロフェッショナルな印刷に最適な形式です。 300 DPI 以上の PNG は、デジタル用途、ドキュメント、Web 表示に最適です。 PDF は直接印刷のワークフローに便利です。
印刷されたバーコードにはどの DPI が必要ですか?
信頼性の高いスキャンを実現するには、最低 300 DPI。バーの幅が狭い高密度バーコードの場合は、600 DPI を使用します。 200 DPI 未満のバーコードは決して印刷しないでください。
JPEG バーコードを印刷に使用できますか?
いいえ、JPEG 非可逆圧縮によりバーコードのバーの端がぼやけ、スキャンの失敗が発生します。バーコードには常に SVG、PNG、または PDF を使用してください。
PNG バーコードをスケールアップできますか?
いいえ。ラスター PNG を拡大するとピクセル化が発生し、バーのエッジがぼやけます。必要な正確な解像度で PNG を生成するか、品質を損なうことなく拡大縮小できる SVG を使用します。
ラベルプリンターにはどのような形式で送信すればよいですか?
最高品質を得るには SVG を送信してください。ラベル メーカーが PDF を必要とする場合は、ベクター PDF を受け入れるかどうかの確認をリクエストしてください。ラスター形式のみが受け入れられる場合は、600 DPI 以上の PNG を指定してください。
