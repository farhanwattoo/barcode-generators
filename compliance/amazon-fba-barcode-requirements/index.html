Amazon FBA バーコード要件 — FNSKU 生成およびラベル配置ポリシー

導入

このガイドでは、Amazon が FNSKU ラベルを必要とする場合、準拠した FNSKU バーコードを生成する方法、受信拒否やチャージバックを回避するためにラベルをレイアウト、印刷、検証する方法を正確に説明します。この本は、Amazon フルフィルメント センターへの商品発送を準備する運用チーム、小規模ブランド オーナー、および契約梱包業者向けに書かれています。

FNSKU が必要になるのはどのような場合ですか?

Amazon は、販売者固有の在庫識別子として FNSKU を使用します。次の場合に FNSKU を使用します。

  • あなたのオファーは、Amazon がその ASIN に対して受け入れるメーカー提供の GTIN にリンクされていません。
  • アカウントまたは出品には、混合（通常はプライベートブランドや中古品）を防ぐために Amazon レベルの追跡が必要です。
  • Amazon は、出荷品の作成または梱包の削除指示の際に、販売者が適用する FNSKU ラベルを明示的に要求します。

注: GTIN が使用されている場合でも、Amazon は特定のプログラムまたはカテゴリで販売者が適用したラベルを受け入れたり、要求したりする場合があります。最新の手順については、セラーセントラルの出荷作成ワークフローを常にご確認ください。

FNSKUラベルのレイアウトと寸法

必須の要素

  • サポートされているシンボル (通常はワークフローごとに指定された Code 128 または UPC) を使用した機械可読バーコード (FNSKU)。
  • 人間が判読できる FNSKU 文字列がバーコードのすぐ下に印刷されます。
  • オプション: 商品タイトルと条件テキスト (Amazon がカテゴリに必要とする場合)。
  • バーコードの周囲に適切なクワイエット ゾーンを設けます (下記のサイズを参照)。

バーコードのシンボルとサイズ設定

Amazon は一般的な線形シンボルを受け入れます。多くの販売者は、英数字文字列とコンパクトな幅をサポートしているため、FNSKU 出力にコード 128 を使用しています。信頼性の高いスキャンのために、Code 128 FNSKU ラベルの推奨デフォルト X 寸法は 0.254 mm (0.010 インチ) です。小さいラベルの場合、バーコードの最小推奨高さは 0.5 インチですが、セラーセントラルのカテゴリ固有のガイダンスに従ってください。

クワイエット ゾーン、マージン、および印刷許容差

Code 128 出力の左右両側に少なくとも 10 × X のクワイエット ゾーンを維持します。たとえば、X=0.254 mm の場合、クワイエット ゾーンは約 2.54 mm (~0.1 インチ) になります。ラベルをエッジ、曲線、継ぎ目、または折り目の上に配置しないでください。ラスタースケーリングアーティファクトを防ぐために、Amazon が要求する正確な物理サイズでベクター出力 (SVG/EPS) を使用します。

プリセットとベクトル出力をエクスポート

プリンタ間での線幅の忠実性を保証するために、可能な場合は常に FNSKU ラベルをベクター ファイル (SVG または EPS) としてエクスポートしてください。推奨されるエクスポート プリセット:

  • SVG: ラスタライズなし、ミリメートル単位、CSS またはコメントでエンコードされた正確な X 次元。
  • PDF (ベクター): フォントを埋め込み、ページ サイズを最終的なステッカーのサイズに設定します。
  • PNG: プルーフのみ。スケーリング エラーを避けるために、最終的な物理サイズで 600 ～ 1200 DPI でエクスポートしてください。

ラベル製造バッチの追跡に役立つように、バーコード クワイエット ゾーンの外側の印刷可能領域に人間が判読できる制御文字列と一意のジョブ ID を含めます。

混合防止とブランド管理

混合を避け、在庫の完全性を保護するには:

  • 共有 ASIN でのプールを防止したい場合は、販売者ごとに一意の FNSKU を使用してください。
  • 可能な場合は Amazon ブランドレジストリに登録して、出品コンテンツとブランド保護をより詳細に制御できるようにします。
  • 印刷されたラベルの保管管理文書を厳密に保管し、ラベル印刷に安全なバッチ番号を適用します。

商品にすでにメーカー GTIN が設定されている場合は、FNSKU の必要性を最小限に抑えるために、出荷前に所有権を確認し、セラーセントラルで競合する出品を解決してください。

チェックリストと検証

  1. Amazon が出荷作成ワークフローで ASIN ごとに FNSKU を必要とするかどうかを確認します。
  2. セラーセントラルから正確な文字列として FNSKU を生成します (手動で転記しないでください)。
  3. 必要な正確な物理的寸法でラベルを SVG/PDF としてエクスポートします。
  4. ターゲット プリンターで校正刷りを印刷し、代表的なスキャナーでハンドヘルドおよび固定マウントのスキャンを実行します。
  5. クワイエット ゾーン、バーコードの高さ、人間が判読できるテキストの配置を確認します。
  6. 継ぎ目、曲線、ヒートシール部分を避けてラベルを梱包して貼り付けます。

よくある問題: 光沢のあるコーティングされた表面上の熱転写による低コントラストのプリント、にじみによるクワイエット ゾーンの切り詰め、手動で入力された不正な FNSKU 値。
